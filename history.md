### Known limitations

- **CoppeliaSim is NOT bundled in the image**. It is mounted as a host-side volume at `/coppeliasim`. PyRep's `setup.py` reads `$COPPELIASIM_ROOT/system/usrset.txt` at install time, so PyRep installation is deferred from Dockerfile to entrypoint.
- **No top-level `peract` package**. `setup.py` uses `find_packages()` which picks up `helpers/`, `agents/`, `voxel/`. `import peract` does not work; use any sub-package for smoke tests.
- **`LD_LIBRARY_PATH=/coppeliasim`** must be set in `docker-compose` environment (not only in `entrypoint.sh`). Exports in the entrypoint do not reach subsequent `docker exec` shells.
- **pytorch3d 0.3.0 requires `--no-build-isolation`** so it can see the already-installed `torch==1.7.1`. The Dockerfile handles this explicitly.
- **RLBench `requirements.txt` is installed separately** (`uv pip install -r /opt/third_party/RLBench/requirements.txt`) because `uv pip install -e <pkg>` does not read `requirements.txt`, only `setup.py install_requires`.
- **Python 3.8** is pinned (not the skill default 3.10) because `torch 1.7.1+cu110` wheels stop at cp38.
- **No `uv.lock`**: this repo uses plain `setup.py` + `requirements.txt`. The image follows the `NO_UV_LOCK=1` variant of the IL template — `uv pip install -r requirements.txt` rather than `uv sync --frozen`.
