# peract — Install Recipe

Minimum commands to reproduce this benchmark container from a fresh clone. Generated by `benchmark-env-generator` skill (sub-skill of `env-generator`).

## Host prerequisites

- Docker (>= 24) and the Compose v2 plugin
- NVIDIA driver + NVIDIA Container Toolkit with a GPU visible to `nvidia-smi`
- CoppeliaSim V4.1.0 Player (Ubuntu 20.04, ~63 MB, free). Install it locally first:

```bash
bash scripts/install_coppeliasim.sh
```

This drops CoppeliaSim into `third_party/CoppeliaSim/` (inside this repo) and writes the `usrset.txt` + `libcoppeliaSim.so.1` symlink that PyRep expects. The compose files default to that path, so `COPPELIASIM_ROOT` does not need to be exported unless you installed elsewhere.

## Build the image

```bash
docker compose -f docker/docker-compose.headless.yaml build
```

Base image: `nvidia/cuda:11.0.3-cudnn8-devel-ubuntu20.04`
Python: `3.8` (managed by `uv` in `/opt/venv`)
Install strategy: `uv pip install -r requirements.txt` (no `uv.lock` — this repo uses classic `setup.py` + `requirements.txt`)

The first build takes roughly 15–20 min because `pytorch3d==0.3.0` is compiled from source against torch `1.7.1+cu110` with `TORCH_CUDA_ARCH_LIST="7.0;7.5;8.0+PTX"`. Subsequent builds hit the cache.

## Bring the container up

```bash
docker compose -f docker/docker-compose.headless.yaml up -d
docker exec -it peract_container bash
```

For an X11 / GUI session (swap the compose profile — container name is shared, only one may run):

```bash
xhost +local:docker
docker compose -f docker/docker-compose.x11.yaml up -d
```

## First-run sanity checks (inside the container)

```bash
nvidia-smi
python -c "import torch; print(torch.__version__, torch.cuda.is_available())"
# peract has no top-level `peract` package — `setup.py` uses `find_packages()`
# which picks up `helpers/`, `agents/`, `voxel/`. Import any of them to verify
# the editable install succeeded:
python -c "import helpers; print(helpers.__file__)"
# expected: /workspace/helpers/__init__.py

# Third-party stack (CoppeliaSim must be mounted for these two):
python -c "import pyrep"
python -c "from rlbench import CameraConfig; print('ok')"
```

The `helpers.__file__` path must resolve under `/workspace/` — that proves the mounted source is what the interpreter imports.

## Category-specific smoke run (IL — WebSocket random-policy loop)

The generated image ships with a companion server so the stack is end-to-end runnable without a trained policy:

```bash
# Start the random-policy server (advertises action_dim=8)
docker exec -d peract_container python tests/test_random_policy_server.py --port 8000

# Wait for the port
docker exec peract_container python -c "import socket, time
for _ in range(20):
    try: socket.create_connection(('127.0.0.1', 8000), timeout=1).close(); print('ready'); break
    except OSError: time.sleep(0.5)"

# Single-episode demo
docker exec peract_container python scripts/run_demo.py \
    --policy_server_addr localhost:8000 --n-steps 10 --task open_drawer

# Eval harness (N episodes, success-rate aggregation)
docker exec peract_container python scripts/run_eval.py \
    --policy_server_addr localhost:8000 --n-episodes 1 --task open_drawer

# Clean up
docker exec peract_container pkill -f test_random_policy_server
```

The random policy never solves the task — a zero success rate plus a clean exit is the expected outcome.

## Real eval (checkpoint + dataset required, not part of smoke)

```bash
# One-time: download a pretrained agent and generate a small validation set
cd /workspace && sh scripts/quickstart_download.sh

cd /opt/third_party/RLBench/tools && python dataset_generator.py \
    --tasks=open_drawer --save_path=/workspace/data/val \
    --image_size=128,128 --renderer=opengl \
    --episodes_per_task=10 --processes=1 --all_variations=True

cd /workspace && CUDA_VISIBLE_DEVICES=0 python eval.py \
    rlbench.tasks=[open_drawer] rlbench.task_name='multi' \
    rlbench.demo_path=/workspace/data/val \
    framework.logdir=/workspace/ckpts/ \
    framework.eval_episodes=10 rlbench.headless=True
```
