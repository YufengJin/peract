# peract — Benchmark Guide

Generated by `benchmark-env-generator` skill. English-only by contract; regenerated on every re-run.

## About

PerAct (Perceiver-Actor) is a language-conditioned behavior-cloning benchmark for multi-task robotic manipulation, introduced by Shridhar, Manuelli, and Fox (*CoRL 2022*). It evaluates a single voxel-based transformer agent across a curated suite of RLBench tabletop manipulation tasks. Each task comes with natural-language variations (e.g. "open the top drawer" vs "open the middle drawer"), and the agent must select a discrete 6-DoF keyframe action (end-effector pose + gripper) conditioned on a CLIP-encoded instruction and a voxel grid assembled from 4 RGB-D cameras. The benchmark is designed for offline imitation learning — there is no shaped reward and no online RL loop.

## Tasks

### Task groups

The 18 tasks cluster loosely by motor skill:

- **Prehension** — `close_jar`, `meat_off_grill`, `stack_blocks`, `stack_cups`, `place_cups`, `place_wine_at_rack_location`
- **Articulated objects** — `open_drawer`, `turn_tap`, `light_bulb_in`
- **Tool use** — `sweep_to_dustpan_of_size`, `reach_and_drag`
- **Insertion** — `insert_onto_square_peg`, `place_shape_in_shape_sorter`
- **Compound (multi-step)** — `put_item_in_drawer`, `put_money_in_safe`, `put_groceries_in_cupboard`
- **Pushing / sliding** — `slide_block_to_color_target`, `push_buttons`

Each task ships with several natural-language variations (colour, location, count, object identity). The full variation set is loaded automatically via `rlbench.all_variations=True` when generating demos — see `install.md` for the dataset-generator command.

## Action / Observation spaces

| Field | Value |
|-------|-------|
| Action dim | `8` (3 position + 4 quaternion + 1 discrete gripper) |
| Action semantics | End-effector pose target consumed by `EndEffectorPoseViaPlanning` + discrete gripper open/close. Real PerAct inference predicts a voxel keyframe; the 8-D action is the planner-level output fed to `task.step()`. |
| Observation type | 4-view RGB-D (front, left shoulder, right shoulder, wrist) @ 128×128 + proprioception (joint positions, gripper open state). PerAct inference voxelises all four views into a `100³` workspace voxel grid. |
| Control frequency | Planner-level (`MoveArmThenGripper` schedules each keyframe as a motion-plan segment; in-between wall-clock depends on planner iterations, not a fixed Hz). |
| Reward signal | Sparse: `reward = 1.0` on task success, `0.0` otherwise. No shaped reward. |

## How to use

The skill produced three receipt files at the repo root — `install.md` (how to build), `history.md` (what the skill did), and this file (what the benchmark is). The flow below assumes you have already followed `install.md` and have a running container.

### 1. Enter the container

```bash
docker compose -f docker/docker-compose.headless.yaml up -d
docker exec -it peract_container bash
```

### 2. Inspect the task registry

```bash
# List RLBench task class names (matches the task ids above modulo snake/camel conversion)
python -c "import rlbench.tasks as t; print([x for x in dir(t) if x[0].isupper()][:20])"
```

Any snake_case id from the task table is a valid value for `--task` on the generated clients — the loader converts it to the corresponding RLBench class name.

### 3. Stand up a policy server

For a smoke test, use the companion random-policy server (ships with the image):

```bash
python tests/test_random_policy_server.py --port 8000
```

For real evaluation, replace it with any server that speaks the `policy_websocket` protocol and advertises `action_dim=8` in its metadata handshake — the client's guard in `scripts/run_eval.py` enforces the match. A minimal trained-policy server wraps the PerAct agent's `act()` method; see `tests/test_random_policy_server.py` for the skeleton.

### 4. Run a single-episode demo

```bash
python scripts/run_demo.py --policy_server_addr localhost:8000 \
    --task open_drawer --n-steps 20
```

Per-step log line shows `reward` and `terminate`. Expect `success_rate=0` under the random policy — the smoke test validates the wire protocol, not the policy.

### 5. Run multi-episode eval

```bash
python scripts/run_eval.py --policy_server_addr localhost:8000 \
    --task open_drawer --n-episodes 10 --max-steps 200
```

Prints `mean_return` and `success_rate` across the rollouts.

### 6. Real training (off-line BC)

For actual PerAct training on this benchmark:

1. **Download demonstrations**: `cd /workspace && sh scripts/quickstart_download.sh` pulls a pretrained checkpoint; for your own demos, generate with the RLBench dataset generator:

   ```bash
   cd /opt/third_party/RLBench/tools && python dataset_generator.py \
       --tasks=open_drawer --save_path=/workspace/data/train \
       --image_size=128,128 --renderer=opengl \
       --episodes_per_task=100 --processes=1 --all_variations=True
   ```

2. **Train**: use the repo's own training entry point (not generated by this skill — BC training is benchmark-specific):

   ```bash
   cd /workspace && python train.py \
       rlbench.tasks=[open_drawer,close_jar,stack_blocks] \
       rlbench.task_name='multi' \
       rlbench.demo_path=/workspace/data/train \
       framework.training_iterations=500000
   ```

3. **Serve + eval**: wrap the trained PerAct agent in a `policy_websocket` server (mirror the structure of `tests/test_random_policy_server.py` but call the agent's `act()` method instead of returning a random vector), then point `scripts/run_eval.py --policy_server_addr` at it.

See upstream `README.md` § "Training & Evaluation" for the full training recipe and hyperparameters.

## Troubleshooting

- `ConnectionRefused` on `run_eval.py` → no policy server on the configured port. Start `tests/test_random_policy_server.py` first, or check your trained-policy server process logs.
- `ValueError: server action_dim=... != expected 8` → the policy server advertises a different action dim. Fix the server's metadata to `{"action_dim": 8}`.
- `pyrep.errors.ConfigurationError: CoppeliaSim not found` → CoppeliaSim volume not mounted. Run `bash scripts/install_coppeliasim.sh` on the host and `docker compose down && up -d` to re-mount.
- Task name not recognised → compare against the 18-row table above; the loader expects exact snake_case.
- GUI apps fail with `cannot open display` → you are on the headless compose profile. Switch to `docker/docker-compose.x11.yaml` after `docker compose down`.
- OpenGL / EGL errors on visual cameras → real PerAct inference voxelises RGB-D from 4 cameras; headless CoppeliaSim + `QT_QPA_PLATFORM=offscreen` cannot create an OpenGL context. For smoke tests we skip cameras (low-dim obs only). Real eval needs the X11 profile or a working EGL/OSMesa path.
